# 🚀 快速开始指南

> 本指南将帮助您在微信开发者工具中打开项目并完成必要的配置。

---

## 📋 前提条件

- ✅ 已注册微信小程序账号
- ✅ 已安装微信开发者工具
- ✅ 已开通云开发环境
- ✅ 获得了 AppID 和云环境ID

---

## 步骤一：打开项目

### 1. 启动微信开发者工具

### 2. 导入项目
- 点击 **"导入项目"** 或 **"+"**
- **项目目录**：选择 `F:\mcp\claude\TimeRecord`
- **AppID**：填写您的 AppID
- 点击 **"导入"**

### 3. 等待项目加载
第一次打开可能需要 10-30 秒，请耐心等待。

---

## 步骤二：配置项目

### ⚠️ 必须修改的配置

#### 1. 修改 AppID

**文件位置：** 根目录 `project.config.json`

找到第 47 行：
```json
"appid": "你的AppID"
```

**修改为：**
```json
"appid": "wxXXXXXXXXXXXXXXXX"  // 您的真实 AppID
```

---

#### 2. 修改云环境ID

**文件位置：** `miniprogram/app.js`

找到第 9 行：
```javascript
env: '你的云环境ID',
```

**修改为：**
```javascript
env: 'timerecord-xxx',  // 您的真实云环境ID
```

**如何查看云环境ID？**
- 在开发者工具中点击顶部 **"云开发"** 按钮
- 在云开发控制台顶部可以看到环境名称和ID
- 格式类似：`timerecord-xxxxx` 或 `prod-xxxxx`

---

## 步骤三：编译运行

### 1. 保存修改
确保上面两处修改已保存（Ctrl+S）

### 2. 点击编译
点击开发者工具顶部的 **"编译"** 按钮

### 3. 查看效果
- 左侧模拟器应该显示首页
- 可以看到实时时间显示
- 中央有一个大的录音按钮

---

## 步骤四：初始化云数据库（首次运行）

### 1. 打开云开发控制台
点击顶部工具栏的 **"云开发"** 按钮

### 2. 创建数据库集合
在左侧菜单选择 **"数据库"**，然后：

#### 创建 `records` 集合
- 点击 **"添加集合"**
- 集合名称：`records`
- 点击 **"确定"**

#### 创建 `tags` 集合
- 点击 **"添加集合"**
- 集合名称：`tags`
- 点击 **"确定"**

#### 创建 `summaries` 集合
- 点击 **"添加集合"**
- 集合名称：`summaries`
- 点击 **"确定"**

#### 创建 `users` 集合
- 点击 **"添加集合"**
- 集合名称：`users`
- 点击 **"确定"**

#### 创建 `templates` 集合
- 点击 **"添加集合"**
- 集合名称：`templates`
- 点击 **"确定"**

### 3. 设置权限
对于每个集合，点击右侧的 **"权限设置"**：
- 选择 **"仅创建者可读写"**
- 点击 **"确定"**

---

## 步骤五：测试功能

### 1. 测试录音功能
- 点击中央的录音按钮
- 会提示请求录音权限，点击 **"允许"**
- 再次点击开始录音
- 再次点击停止录音

### 2. 查看控制台
- 打开开发者工具底部的 **"调试器"** 标签
- 选择 **"Console"**
- 可以看到录音相关的日志信息

---

### 2. 扫码体验

## ❓ 常见问题

### Q1: 编译失败，提示"找不到 xxx 页面"
**解决：** 检查 `miniprogram/app.json` 中的 pages 配置是否正确

### Q2: 云开发按钮点不了
**解决：**
1. 确认 AppID 不是测试号
2. 确认已在微信公众平台开通云开发

### Q3: 提示"云环境初始化失败"
**解决：**
1. 检查 `app.js` 中的环境ID是否正确
2. 在云开发控制台确认环境已开通

### Q4: 录音没反应
**解决：**
1. 检查是否授权了录音权限
2. 真机预览时才能真正录音，模拟器只能模拟

### Q5: 底部 tabBar 图标不显示
**解决：**
- 这是正常的，图标文件还未添加
- 暂时不影响功能测试
- 后续会添加图标资源
